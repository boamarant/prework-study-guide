HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for a webpage.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain properties and values, which define the type of style we want to apply as well as the value of the style.
  • For example, "color: blue;" defines the property as "color" and the value of the color as "blue".
  • The link element can connect CSS files to other types of files, such as HTML.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • The add command adds modification s in the current working branch to the staging area.
  • git commit: describes what the commit contains.
  • The pull command receives a branch's modifications into the local environment.
  • The push command pushes the local changes to the GitHub repo.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • If statements will check if a statement is truthy, and perform a function if its conditions are met.
  • A for loop will check until a condition is met to perform a function and will not proceed until it is.
  • Arrays are used to store multiple variables so that functions like for loops can check all of them
  • The first number in an array's index is 0 rather than 1, and items can be checked in the index linearly
